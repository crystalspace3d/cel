Detailed Description

This interface is implemented by the trigger that fires when a mesh becomes visible.

You can query this interface from the trigger factory if you want to manually control this factory as opposed to loading its definition from an XML document.

The predefined name of this trigger type is 'cel.questtrigger.watch'.

In XML, factories recognize the following attributes on the 'fireon' node:

  • entity: the name of the source entity that is watching out for other entity. This should contain a pcmesh to get a position from.
  • entity_tag: optional tag used to find the right property class from the entity.
  • target: the name of the target entity to watch. This should contain a pcmesh to get a position from.
  • target_tag: optional tag used to find the right property class from the target entity.
  • checktime: optional tag to specify the check interval.
  • radius: optional tag to specify the maximum radius.
